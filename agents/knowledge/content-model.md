# Content model

## Models live in `BlogWebApp/Models/`

| Model | Container | Notes |
|---|---|---|
| `BlogPost` | `Posts` | The core content document. |
| `BlogPostComment` | `Posts` | Inline doc on the same partition as its parent post. **Not surfaced in v1** (comments are a non-goal — spec §9). Data layer kept for change-feed compatibility. |
| `BlogPostLike` | `Posts` | Same pattern as comment. **Not surfaced in v1.** |
| `BlogUser` | `Users` | The pre-Identity user shape. Phase 1c introduces `CosmicBlogUser` with `PasswordHash`, `Email`, `EmailNormalized`, `SecurityStamp`, `Roles[]`. |
| `BlogUniqueUsername` | `Users` | Helper for the unique-key constraint on `/username`. |

## `BlogPost` shape — current

```csharp
public class BlogPost
{
    public string Id          { get; }              // == PostId (derived getter; Cosmos requires "id")
    public string PostId      { get; set; }
    public string Type        { get; set; } = "post";  // valid: "post" | "note" | "now"
    public string Slug        { get; set; } = string.Empty;
    public string? LinkUrl    { get; set; }             // notes only
    public DateTime? DateUpdated { get; set; }
    public string Status              { get; set; } = "published";  // "draft" | "published"
    public DateTime? PublishedAtUtc   { get; set; }                  // future = scheduled
    public string Format              { get; set; } = "markdown";    // "markdown" | "html" (legacy)
    public List<string> Tags          { get; set; } = new();
    public string? Excerpt            { get; set; }
    public string? CoverImageUrl      { get; set; }
    public string AuthorId    { get; set; }         // -> "userId" in JSON (legacy naming)
    public string AuthorUsername { get; set; }      // -> "userUsername"
    public string Title       { get; set; }
    public string Content     { get; set; }         // markdown source (new) or HTML (legacy); dispatched via IMarkdownRenderer on Format
    public int    CommentCount { get; set; }        // not surfaced in v1
    public int    LikeCount    { get; set; }        // not surfaced in v1
    public DateTime DateCreated { get; set; }
}
```

JSON property names use Newtonsoft `[JsonProperty(PropertyName = "...")]`. `Id` is a derived getter that returns `PostId` — Cosmos requires an `id` field; the project uses `postId` as the partition key and the derived `id` simply re-exposes it.

`CommentCount` / `LikeCount` survive in the schema for change-feed compatibility but are not read or written from the public surface.

## Markdown rendering

Content is stored as either markdown source (`Format = "markdown"`) or rendered
HTML (`Format = "html"`, for pre-Phase-1d content). `IMarkdownRenderer.Render(content, format)`
dispatches: markdown → Markdig (with advanced extensions); html → emitted as-is.
Views inject `IMarkdownRenderer` and call `Render` instead of `@Html.Raw`.

## Content types and URL conventions (post-Phase-1d)

| `Type` | Public URL | Admin URL | Notes |
|---|---|---|---|
| `post` | `/posts/{slug}` | `/admin/posts/new`, `/admin/posts/edit/{postId}` | Long-form. Slug is mandatory and immutable across edits. |
| `note` | `/notes/{slug-or-id}` | `/admin/notes/new`, `/admin/notes/edit/{postId}` | Title optional; if title is empty, URL falls back to `postId` GUID. |
| `now` | `/now` | `/admin/now` | Single-document page at fixed `postId = "now-singleton"`. |
| tag listing | `/tag/{tag}` | (none) | Public cross-type listing of posts + notes that ARRAY_CONTAINS the tag. |
| admin lists | (none) | `/admin/posts`, `/admin/notes` | Filterable by status (All / Drafts / Scheduled / Published). |

## Slug rules

Generated by `Services/SlugGenerator.FromTitle(string)`:

1. Unicode normalize → strip diacritics → ASCII.
2. Lowercase.
3. Replace runs of non-`[a-z0-9]` with `-`. Trim leading/trailing `-`.
4. Cap at 80 chars.

Slug is generated **on create only** and held stable across edits. On collision, append the first 8 chars of the new `postId` GUID. Don't change the slug generation logic without a migration plan for existing slugs.

## Now singleton

Phase 1c stores the Now page as a single Cosmos document:

- `postId = "now-singleton"` (fixed)
- `type = "now"`
- partition key value = `"now-singleton"`

`IBlogCosmosDbService.GetNowAsync()` and `UpsertNowAsync()` enforce the singleton id — callers don't manage it.

## Cosmos partition key choices

- **`Posts` /postId** — every doc is its own partition. Cheap for point reads/writes, requires cross-partition queries for activity feeds (acceptable at v1 scale of dozens of posts). Phase 1c stays on this partition key for backward compat; the spec acknowledges container topology could be revisited later.
- **`Users` /userId** — same one-doc-per-partition pattern.
- **`Feed` /type** — designed for the homepage cross-type query. Post-Phase-1c, this container becomes legacy for the homepage path (queries hit Posts directly).

## What's NOT in the content model

Phase 1d does NOT add:
- Post revisions / version history — out of scope.
- Per-tag entities with description / metadata — tags are simple strings on the post.
- Multi-language content — single-language v1.
